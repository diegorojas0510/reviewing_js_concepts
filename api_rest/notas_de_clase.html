¿Què es una API

  • Api: Aplication program interface / mecanismos con que los usuarios pueden interacturar a travéz de bótones menús imagenes click etc..

  • ¿Qué es Una API REST?

    • Api Rest: Este es un mecanismo de interacción de robot con otros robots ejemplo backend con backend o backend con el frontend

    • Rest: Representecional state transfer Api Rest en definidas cuentas es conexión de apis por medio de internet http el cual no va a responder en formato json

    • Flujo de Api rest En la siguiente imagen podemos ver como nos conectamos desde una pagina web, hacemos peticiones hacia el servidor y este nos responde con html pero para ser más interactivo tambien utilizamos js para recibir peticiones enviarlar via api manipular el DOM con su respuesta y para que persista la información lo hacemos mediante la BD.

  • Reconociendo mi url de API

    • HTTPS = Protocolo de transferencia de datos de manera segura (Protocolo de Transferencia de Hipertexto Seguro)
    • Host y Endpoint api.thedogapi.com este es el servidor al que se le hace la solicitud
    • Ruta v1/images/search: este enpoint es la ruta que maneja la solicitud especifica
    • ?: indica el inicio de de lo párametros de consulta en este caso limit=10
    • Limit=10: Este párametro limita el número de resultados que se devuelven en la respuesta de la api cantidad de perros 10
  • Http Status Code
  • centena de 200 Front y Back esta bien
  • 201 Creación Bien
  • 202 Creación bien pero falta
  • Centena de 300
  • Redirect Nuestro back nos envia a donde le decimos
  • 307 redirect no sabemos si esa reta es la misma ruta donde nos envian siempre que consultamos el sitio
  • 308 redirect es el backend diciendonos siempres que lleges aquí siempre te voy a enviar a esta ruta
  • Centenas del 400 Error por parte de frontend quiere decir que hicimos una petición incorrecta sintaxis o algo en la ruta
  • 401 el endpoint no funciona de esa forma a la que nosotros queremos acceder
  • 402 necesitamos pagar para acceder y recibir la información
  • 404 lo que nosotros estamos solicitando no existe
  • Centenas del 500 el backend falló
  • Nada que hacer desde el frontend
  • Autenticación Saber quien es usted?
  • Autorización que puedes hacer dentro(Permisos)
  • header de Autenticación
